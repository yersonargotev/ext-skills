# Use `@grupo-exito/harness` as the package name

We will publish the installer CLI as `@grupo-exito/harness`. The unscoped `harness` package is already taken on npm, and the `@harness/*` scope is attractive but uncertain to reserve; `@grupo-exito/harness` keeps the product name while making ownership and internal branding explicit.

## Consequences

The CLI command is slightly longer when invoked through npx, but the package identity is clear and safer for an Éxito-owned tool.
